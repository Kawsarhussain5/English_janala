Frequently Asked Questions

The difference between var, let, and const
1.var is function-scoped, while let and const are block-scoped, meaning they are only accessible within the block where they are defined.
2.let and const help prevent accidental redeclaration and provide better scoping than var, reducing potential bugs.
3.const is used for values that should not change, while let is preferred for variables that may be updated later.
The difference between map(), forEach(), and filter()
1.map() creates a new array by applying a function to each element, returning a modified version of the original array.
2.forEach() executes a function on each array element but does not return a new array; it simply iterates over the elements.
3.filter() creates a new array containing only the elements that pass a specified condition.
Explain arrow functions and how they are different from regular functions
Arrow functions are a concise way to write functions in JavaScript using the => syntax. They provide a shorter and more readable alternative to regular function expressions.
How JavaScript Promises work
A Promise in JavaScript is an object that represents the eventual completion (or failure) of an asynchronous operation. It allows handling asynchronous code more efficiently than using callbacks.
how closures work in JavaScript
A closure is a function that retains access to variables from its lexical scope (the scope where it was created) even after the outer function has finished executing..
